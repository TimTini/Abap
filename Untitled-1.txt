Ngoài ra đối với các đặt tên biến.
Hãy giúp tôi chuẩn hóa lại cho thống nhất, vì có thể trong một program được chỉnh sửa nhiều lần, cách đặt tên cũng lẫn lộn.
Với các tiền tố.
1 ký tự ngẫu nhiên + `CN` thì hãy tự động đặt lại tên là => HẰNG:[Mô tả từ user|code]
1 ký tự ngẫu nhiên + `DS` thì hãy tự động đặt lại tên là => STRUCT:[Mô tả từ user|code]
1 ký tự ngẫu nhiên + `DT` thì hãy tự động đặt lại tên là => TABLE:[Mô tả từ user|code]
1 ký tự ngẫu nhiên + `DR` thì hãy tự động đặt lại tên là => RANGETABLE:[Mô tả từ user|code]
1 ký tự ngẫu nhiên + `DF thì hãy tự động đặt lại tên là => BIẾN:[Mô tả từ user|code]
1 ký tự ngẫu nhiên + `FL` thì hãy tự động đặt lại tên là => CỜ:[Mô tả từ user|code]
1 ký tự ngẫu nhiên + `FS` thì hãy tự động đặt lại tên là => FIELDSYMBOL:[Mô tả từ user|code]
Các trường hợp bên trên hãy cho vào modal để user có thể chỉnh sửa riêng.

Trường hợp Mô tả của USER|Code đã bắt từ "Hằng", "STRUCT", "TABLE", ... thì chỉ cần format lại phía sau cho thống nhật.
Tránh bị trùng.

Kiểm tra và Update phần lấy mô tả từ code. Đang có lỗi xảy ra.
Trường hợp khi dùng nhiều câu lệnh cùng lúc (đùng dấu :), thi mô tả của nó bị cộng dồn.
Ví dụ:
`* TYPES (BEGIN/END) + plain TYPES
TYPES: BEGIN OF ty_row,
         bukrs TYPE t001-bukrs, "Mã Công ty
         butxt TYPE t001-butxt, "Tên Công ty
       END OF ty_row.
TYPES ty_rows TYPE STANDARD TABLE OF ty_row.

* DATA / CONSTANTS / RANGES / FIELD-SYMBOLS / STATICS
DATA gv_user TYPE syuname. "Global user
DATA: gt_rows TYPE ty_rows, "Thông tin công ty
      gs_row  TYPE ty_row,  "Danh sách thông tin công ty
      gv_cnt  TYPE i VALUE 0,
      lo_demo TYPE REF TO lcl_demo,
      lv_text TYPE string.`


Ở khai báo data có dùng ":"
Kết quả lấy mô tả từ code hiện tại sẽ là "Thông tin công ty Danh sách thông tin công ty"
Nó đang bị nối lại với nhau cho toàn bộ data được khai báo. Như v là không đúng.
Đúng phải là:
    Chỉ gt_rows sẽ có mô tả là "Thông tin công ty"
    Chỉ gs_row sẽ có mô tả la "Danh sách thông tin công ty"
    gv_cnt, lo_demo, lv_text không có mô tả.
